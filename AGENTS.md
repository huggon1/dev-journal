# AGENTS.md

## Collaboration

- GitHub is the source of truth for issues, branches, pull requests, and review.
- Each development branch should correspond to one GitHub issue.
- A session should focus on one issue, one branch, or one clear phase of work.
- Simple issues may be completed end-to-end in one session.
- Complex issues may be split across planning, implementation, review, and CI-fix sessions.
- Agents should read this file before making project changes.
- Keep changes scoped to the current issue unless the user explicitly asks otherwise.
